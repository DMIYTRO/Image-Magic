# Установка и тестирование Image Magic Batch Processor

Этот документ содержит полный список зависимостей и порядок запуска пакетного обработчика на другом компьютере.

## 1. Что необходимо перенести

Скопируйте проект целиком либо как минимум следующие файлы и папки, сохранив структуру:

```text
Image Magic/
├── process_orders.py
├── core/
│   ├── __init__.py
│   ├── inspector.py
│   ├── pdf_exporter.py
│   ├── resampler.py
│   ├── report_builder.py
│   └── history_db.py
└── processing/
    ├── __init__.py
    ├── models.py
    ├── filename_parser.py
    ├── batch_processor.py
    └── resample_policy.py
```

Запускать `process_orders.py` необходимо из корневой папки проекта.

## 2. Зависимости

### Обязательные программы

- Python 3.10 или новее;
- ImageMagick с доступной командой `magick`;
- Ghostscript с доступной командой `gs`;
- Python-библиотеки из `requirements.txt` (`SQLAlchemy` и `typing_extensions`).

### Для чего нужны зависимости

- Python запускает обработчик и выполняет группировку заказов.
- ImageMagick читает TIFF/JPEG/PNG, DPI, физический размер и CMYK, определяет количество страниц TIFF и создаёт временные одностраничные PDF.
- Ghostscript объединяет временные PDF в итоговый документ и проверяет его открытие, количество и размеры страниц.
- SQLAlchemy сохраняет историю проверок в базе SQLite.

## 3. Установка на macOS

### Установка Homebrew

Если Homebrew ещё не установлен, используйте инструкцию с официального сайта:

```text
https://brew.sh
```

### Установка зависимостей

```bash
brew install python imagemagick ghostscript
python3 -m pip install -r requirements.txt
```

Проверка:

```bash
python3 --version
magick -version
gs --version
```

## 4. Установка на Ubuntu/Debian

```bash
sudo apt update
sudo apt install python3 imagemagick ghostscript
python3 -m pip install -r requirements.txt
```

Проверка:

```bash
python3 --version
magick -version
gs --version
```

На некоторых версиях Linux команда ImageMagick может называться `identify`, а не `magick`. Чтение метаданных поддерживает оба варианта, но создание PDF требует команды `magick`. Для полного цикла рекомендуется ImageMagick 7.

## 5. Подготовка тестовой папки

Создайте отдельную папку и поместите в неё копии тестовых файлов. Не используйте единственные экземпляры производственных файлов при первом тесте.

Поддерживаемые входные форматы:

```text
TIFF, TIF, JPEG, JPG, PNG
```

Исключённые форматы:

```text
PSD, BMP, HEIC, HEIF
```

Исключённые файлы не обрабатываются и копируются в `Troubles/UNSUPPORTED`.

## 6. Требования к имени файла

Пример корректного имени:

```text
01_NP(K)_(P)_Bezlam_170_mel_gl_(200x140)_4-4_T100_(8717-25516213)_creas1_falc1_offset_1-face.jpg
```

Обязательные элементы:

- размер строго в круглых скобках: `_(200x140)_`;
- цветность;
- клиент и уникальный заказ: `_(8717-25516213)_`;
- окончание стороны: `face` или `back`.

Допустимая цветность:

```text
4-4, 4-0, 1-0, 1-1, 5-5, 6-0, 6-6, 5-0
```

Правила сторон:

- если справа `0`, требуется только `face`;
- если справа число больше нуля, требуются `face` и `back`;
- `face` становится первой страницей PDF;
- `back` становится второй страницей PDF;
- группировка выполняется по уникальному номеру заказа, в примере — `25516213`.

## 7. Параметры автоматической проверки

- к ширине и высоте из имени добавляется 4 мм;
- отклонение до 0,1 мм считается погрешностью метаданных;
- отклонение/обрезка от 0,1 до 0,5 мм автоматически корректируется с предупреждением;
- обрезка от 0,5 до 2 мм требует подтверждения пользователя;
- более существенная обрезка блокируется;
- прямой и переставленный размеры считаются одинаково допустимыми;
- односторонний заказ не поворачивается автоматически по порядку размеров в имени;
- требуется минимум 300 DPI по обеим осям;
- CMYK рекомендуется, RGB/sRGB разрешается с предупреждением без преобразования цвета;
- ориентация `face` и `back` должна совпадать;
- многостраничный TIFF блокируется;
- разные входные форматы сторон разрешены, но отображаются как предупреждение.

Пример:

```text
Размер в имени: 200x140 мм
Ожидаемый файл: 204x144 мм
Допустимая ориентация: 204x144 или 144x204 мм
```

При коррекции изображение масштабируется только пропорционально. Принудительное растягивание сторон запрещено; небольшой избыток обрезается симметрично по центру.

## 8. Команда безопасного первого запуска

Откройте терминал и перейдите в папку проекта:

```bash
cd "/полный/путь/к/Image Magic"
```

Запустите проверку:

```bash
python3 process_orders.py --input "/полный/путь/к/тестовым/файлам"
```

После вывода отчёта появится вопрос:

```text
Создать PDF для подходящих заказов (N)? [y/N]:
```

Для первого теста нажмите Enter. PDF создаваться не будут.

Если размер требует подтверждения, вопросы о коррекции появятся перед общим отчётом. В них указываются целевой размер, предполагаемая обрезка и поворот.

Важно: даже без создания PDF проблемные файлы копируются в папку `Troubles`. Оригиналы не удаляются и не изменяются.

## 9. Полный тест с созданием PDF

Повторите запуск и ответьте `y` или `да`:

```bash
python3 process_orders.py --input "/полный/путь/к/тестовым/файлам"
```

Без дополнительного параметра PDF сохраняются сюда:

```text
<входная папка>/PDF/
```

Чтобы указать отдельную выходную папку:

```bash
python3 process_orders.py \
  --input "/полный/путь/к/тестовым/файлам" \
  --output "/полный/путь/для/готовых/PDF"
```

Существующий PDF с тем же именем перезаписывается только после успешного создания и проверки нового документа.

## 10. Как формируется PDF

Для двустороннего заказа:

```text
face.tif/jpg/png
  -> ImageMagick -> временный одностраничный face.pdf

back.tif/jpg/png
  -> ImageMagick -> временный одностраничный back.pdf

face.pdf + back.pdf
  -> Ghostscript -> итоговый PDF
```

Ghostscript запускается с отключённым уменьшением разрешения, `LeaveColorUnchanged`, выключенным автоповоротом страниц и без повторного JPEG-кодирования. Временные файлы удаляются автоматически.

Имя итогового PDF берётся от исходного файла `face`, меняется только расширение на `.pdf`.

## 11. Проверка готового PDF

До публикации итогового файла обработчик проверяет:

- файл существует и не пуст;
- Ghostscript может открыть и отрисовать документ;
- количество страниц совпадает с заказом;
- первая ожидаемая страница соответствует `face`, вторая — `back`;
- физический размер каждой страницы совпадает с проверенным исходником.

После обработки также создаются превью, интерактивный HTML-отчёт и запись аудита в SQLite.

Если проверка не пройдена, итоговый PDF не сохраняется.

## 12. Папка Troubles

Проблемные файлы копируются сюда:

```text
<входная папка>/Troubles/<номер заказа>/
<входная папка>/Troubles/UNPARSED/
<входная папка>/Troubles/UNSUPPORTED/
```

Рядом с каждой копией создаётся файл:

```text
имя-файла.ext.error.txt
```

В нём содержится причина отказа. Исходный файл остаётся на прежнем месте.

Если входные файлы прошли проверку, но создание или проверка PDF завершились ошибкой, весь заказ также копируется в `Troubles`.

## 13. Быстро изменяемые настройки

В начале `process_orders.py`:

```text
INPUT_DIR          — входная папка по умолчанию
PDF_DIR_NAME       — имя папки готовых PDF
TROUBLES_DIR_NAME  — имя папки проблемных файлов
```

В `BatchProcessor`:

```text
size_extra_mm = 4.0
tolerance_mm  = 0.5
min_dpi       = 300.0
```

Параметры командной строки `--input` и `--output` имеют приоритет над путями по умолчанию.

## 14. Возможные ошибки

### Ghostscript не найден

```text
Ghostscript (`gs`) не найден
```

Проверьте:

```bash
gs --version
```

### ImageMagick не найден

```text
ImageMagick (`magick`) не найден
```

Проверьте:

```bash
magick -version
```

### PDF запрещён политикой ImageMagick

На некоторых Linux-системах политика ImageMagick запрещает запись PDF. Проверьте файл `policy.xml` установленной версии ImageMagick. Изменять системную политику следует только осознанно и с учётом требований безопасности организации.

### Нет прав на запись

Убедитесь, что пользователь может создавать папки `PDF` и `Troubles` внутри входной директории либо укажите доступную папку через `--output`.

## 15. Контрольный сценарий

Для проверки переноса на другое устройство рекомендуется использовать:

1. корректную пару CMYK `face/back`, 300 DPI;
2. односторонний заказ `4-0` с одним `face`;
3. файл RGB — должен пройти с предупреждением без преобразования цвета;
4. файл ниже 300 DPI;
5. файл с неправильным размером;
6. заказ без `back`;
7. пару с разной ориентацией и почти квадратный заказ;
8. многостраничный TIFF;
9. PSD или HEIC;
10. пару TIFF + JPEG для проверки предупреждения о разных форматах.

Ожидаемый результат:

- корректные заказы предлагаются для создания PDF;
- ошибочные заказы копируются в `Troubles` с причинами;
- после подтверждения PDF создаются только для прошедших заказов;
- небольшие коррекции выполняются без деформации;
- односторонний макет не поворачивается из-за переставленных размеров в имени;
- RGB остаётся RGB и отображается как предупреждение;
- исходные файлы остаются неизменными.
