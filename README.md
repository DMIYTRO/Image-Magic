# Image Magic — допечатная проверка и подготовка заказов

Image Magic проверяет растровые макеты перед печатью, группирует `face/back` по номеру заказа, безопасно корректирует небольшие отклонения размеров, создаёт итоговые PDF, превью и интерактивный HTML-отчёт.

Поддерживаемые входные форматы: `TIFF`, `JPEG`, `PNG`. Форматы `PSD`, `BMP`, `HEIC` и `HEIF` исключаются из обработки и направляются в `Troubles` с объяснением причины.

## Возможности

- разбор размера, цветности, клиента, заказа и стороны из имени файла;
- группировка односторонних и двусторонних заказов;
- проверка физического размера, DPI, количества кадров TIFF и цветовой модели;
- минимум 300 DPI после любой коррекции;
- безопасное пропорциональное масштабирование без растягивания сторон;
- центральная обрезка небольшого избытка макета;
- подтверждение спорных коррекций перед созданием PDF;
- RGB/sRGB отображается как предупреждение и сохраняется без преобразования в CMYK;
- одинаково допустимы размеры `50×90` и `90×50` — порядок сторон в имени не задаёт верх макета;
- контроль ориентации `face/back` без автоматического поворота одностороннего заказа;
- итоговая проверка количества, порядка и размера страниц PDF;
- превью с линиями реза и безопасной зоны;
- HTML-отчёт с ошибками, предупреждениями, параметрами коррекции и визуальной проверкой ориентации;
- шаблонные причины возврата из справочника с поиском и множественным выбором отдельно для каждого заказа;
- формирование и копирование готовых комментариев возврата по выбранным заказам;
- копирование отклонённых заказов в `Troubles` без изменения оригиналов;
- сохранение истории проверок в SQLite через SQLAlchemy.

## Политика размеров

Размер изделия берётся из имени файла, после чего к ширине и высоте добавляется 4 мм вылетов.

- отклонение до 0,1 мм включительно считается погрешностью метаданных;
- отклонение/центральная обрезка от 0,1 до 0,5 мм корректируется автоматически с предупреждением;
- центральная обрезка от 0,5 до 2 мм требует подтверждения пользователя;
- более существенная обрезка отклоняется;
- увеличение разрешается только при итоговом эффективном разрешении не ниже 300 DPI;
- независимое растягивание ширины и высоты запрещено.

Коррекция выполняется в режиме `cover`: изображение пропорционально заполняет целевой формат, после чего избыток симметрично обрезается по центру.

## Ориентация

- прямой и переставленный размеры считаются одинаково допустимыми;
- заказ `4-0` не поворачивается автоматически из-за порядка размеров в имени;
- если `face` и `back` ориентированы одинаково, они остаются без поворота;
- квадратные и почти квадратные изделия сначала нормализуются по размеру и не считаются повёрнутыми из-за малой разницы сторон;
- несовпадение ориентаций сторон требует отдельной проверки;
- HTML-отчёт показывает предупреждение и превью для контроля совмещения.

## Цветовая модель

CMYK остаётся рекомендуемой моделью для печати, но RGB/sRGB больше не блокирует заказ. В отчёте появляется предупреждение, а при создании PDF цветовая модель не преобразуется.

Ghostscript объединяет страницы с `ColorConversionStrategy=LeaveColorUnchanged`, отключённым даунсемплингом и выключенным автоматическим поворотом страниц.

## Структура проекта

```text
Image-Magic/
├── process_orders.py               # Полный пакетный процесс
├── processing/
│   ├── batch_processor.py          # Проверка заказов и создание PDF
│   ├── filename_parser.py          # Разбор имён файлов
│   ├── models.py                   # Результаты проверки
│   └── resample_policy.py          # Решение: принять/исправить/спросить/отклонить
├── core/
│   ├── inspector.py                # Метаданные через ImageMagick
│   ├── resampler.py                # Пропорциональная коррекция и обрезка
│   ├── pdf_exporter.py             # Создание и объединение PDF
│   ├── preview_generator.py        # Превью и контрольные рамки
│   ├── report_builder.py           # HTML-отчёт
│   └── history_db.py               # SQLite-аудит
├── tests/                          # Автоматические тесты
├── requirements.txt
└── INSTALL_AND_TEST.md             # Подробная установка и проверка
```

## Установка

На macOS:

```bash
brew install python imagemagick ghostscript
python3 -m pip install -r requirements.txt
```

Проверка зависимостей:

```bash
python3 --version
magick -version
gs --version
```

## Запуск

```bash
python3 process_orders.py --input "/путь/к/папке/с/макетами"
```

Направление выбирается параметром профиля:

```bash
python3 process_orders.py --input "/путь/к/папке/с/макетами" --direction digital
python3 process_orders.py --input "/путь/к/папке/с/макетами" --direction offset
```

Наборы правил находятся в `config/profiles.py`. Новое направление добавляется
как ещё одна запись в `PROFILES`, без дублирования движка проверки.

Во время запуска программа:

1. проверит и сгруппирует файлы;
2. запросит подтверждение для спорных коррекций;
3. предложит создать PDF для подходящих заказов;
4. предложит создать превью;
5. сохранит аудит и HTML-отчёт.

Результаты создаются во входной папке:

```text
PDF/                 итоговые PDF
Previews/            превью страниц
Troubles/            отклонённые файлы и причины
output_report/       интерактивный report.html
```

## Тесты

```bash
python3 -m unittest discover -s tests -v
```

Тесты покрывают пороги размеров, эффективный DPI, подтверждения, сохранение пропорций, RGB-предупреждения, команды ImageMagick/Ghostscript и HTML-контроль ориентации.

## Безопасность обработки

- исходные изображения не изменяются;
- промежуточные файлы создаются во временной папке;
- итоговый PDF публикуется только после контрольного рендеринга;
- при ошибке весь заказ копируется в `Troubles`;
- цветовые пространства не конвертируются автоматически;
- принудительный ImageMagick resize с `!` не используется.

Подробная инструкция: [INSTALL_AND_TEST.md](INSTALL_AND_TEST.md).
